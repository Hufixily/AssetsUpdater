本测试案例基于ABSystem(https://github.com/tangzx/ABSystem)打包系统开发，主要用于更新游戏资源
功能介绍
1.断点续传
2.打包自动处理依赖数据并打包到外部资源路径
3.自动分析外部路径资源，校检最新资源，从资源服务器下载最新的资源

项目资源讲解
1.StreamingAssets里的资源为包内资源，此资源作为资源更新校检的存在，尽量不要删除
2.使用的更新资源扔到有效的资源服务器内

使用方法
1.使用Tool->Build Asset 生成外部资源
2.将外部资源放到有效的资源服务器
3.进入场景Updater
4.设置Updater(GameObject)里的URL，启动

PS:
因为资源更新中已经包含了配置表资源，所以配置表热更的功能，本测试案例可以通过版本号差异来实现配置表热更
所以需要服务器那边通知版本号变更的消息，客户端监听此消息，来达到配置表热更的功能
建议版本号格式为：大版本编号.资源更新编号.配置表更新编号
